# 개인정보처리위탁계약 (DPA·Data Processing Agreement)

> 30-apps 공유 DPA 템플릿·B2B (도서관·재단·자치구) 위탁 시 의무·PIPA §26.
> 각 앱이 사업자 정보·위탁 영역 채워서 사용·법무 자문 별도 권장.

## 1. 계약 당사자

- **위탁자 (Controller)**: __[고객 사업자명·등록번호]__
- **수탁자 (Processor)**: __[우리 사업자명·등록번호]__
- 본 계약 = PIPA §26 (개인정보 처리위탁) 정합·전자상거래법 호환.

## 2. 위탁 처리 영역

### 2.1 위탁자 → 수탁자 제공 정보

| 영역 | 항목 | 보유 기간 |
|---|---|---|
| 회원 식별 | 이메일 (마스킹 표시·mask_email) | 회원 탈퇴 시까지 |
| 결제 식별 | 결제 기록 (전자상거래법 §13·5년) | 5년 |
| 사용 통계 | audit log (해시 체인) | 6개월 |

→ **사용자 핵심 데이터 (사서·프리랜서 데이터) = 사용자 컴퓨터 한정** (헌법 §14·서버 X).

### 2.2 수탁자가 처리하지 않는 정보

- 주민등록번호·여권·운전면허
- 건강·의료 정보
- 위치·생체 정보
- 사상·신념·노조·정치 (민감정보 5종 + α)

→ 수탁자 = 본 정보 수집·저장·전송 X (시스템 차단).

## 3. 처리 목적

1. 회원 인증·로그인·이메일 인증
2. 결제·환불·세금계산서 발급
3. 갱신·취소 알림
4. 사고 대응 (audit log·72h 신고 의무)
5. 서비스 개선 (집계 통계·익명화 후만)

## 4. 안전성 확보 조치 (PIPA §29·시행령 §30)

| 영역 | 조치 |
|---|---|
| 암호화 | TLS 1.2+ (전송)·bcrypt rounds=12 (저장) |
| 접근 통제 | LoginRateLimiter (5회 5분·15분 락) |
| 인증 | email verification token (HMAC·24h 만료) |
| 위·변조 차단 | webhook HMAC + idempotency + order tampering 검증 |
| 무결성 | AuditChain 해시 체인 (SHA-256·PIPA 5/5) |
| 접근권한 | 최소 권한 (PO 1인 운영·subprocessor X) |
| 개인정보처리 시스템 | Streamlit Cloud (Apache-2.0·미국 거점) |
| 백업 | 결제·audit만 (사용자 데이터 X·헌법 §14) |
| 침입 차단 | brute force·tampering·webhook 위조 자동 차단 |

## 5. 위탁자 권리 (감독·통제)

- 본 DPA 미정합 = 위탁자 즉시 해지 가능 (60일 통지)
- 분기별 보고서 = 위탁자 요구 시 제공 (audit chain·접근 로그·KPI)
- 침투 시험 = 위탁자 비용 부담 시 협조
- 사고 발생 = **24시간 내** 위탁자에게 통지 (PIPA 72h 신고 별도 진행)

## 6. Subprocessor (재위탁)

| 영역 | 재위탁 | 정합 |
|---|---|---|
| Streamlit Cloud (호스팅) | ✅ | Apache-2.0·SOC 2 Type II |
| PortOne v2 (결제) | ✅ | 한국 PG·전자세금계산서·ISMS-P |
| Resend (이메일) | ✅ | 미국·SOC 2·SES wrapper |

→ 재위탁 변경 = **30일 사전 공지** + 위탁자 이의 시 협의.

## 7. 사고 대응 (Incident)

### 7.1 통지 의무 (정직)

- 1,000명+ 정보 유출 = **72시간 내** 정보주체 + KISA 신고 (PIPA 시행령)
- 위탁자에게 = **24시간 내** 통지
- "수정 후 통지" 금지 (대법원 판례 = 즉시 통지)

### 7.2 분류

| 등급 | 정의 | 24h 통지 |
|---|---|---|
| Critical | 정보 유출·해킹·시스템 침해 | ✅ |
| High | 결제 차단·로그인 차단 | ✅ |
| Medium | 부분 장애 | 영업일 1일 내 |

## 8. 데이터 이관·삭제

### 8.1 계약 종료 시

- 위탁자 요구 = 30일 내 모든 데이터 표준 형식 (CSV·JSON) 반환
- 반환 후 = 30일 보관 → 영구 삭제 (audit 포함)
- 삭제 증명서 = 위탁자에게 제공

### 8.2 사용자 DSAR (PIPA §35의2)

- 위탁자 = 사용자 요구 위탁 → 수탁자 = 30일 내 처리
- 비용 = 무료 (연 1회·이후 인쇄·매체 비용 청구 가능)

## 9. 손해배상 (Liability)

- 수탁자 과실 시 = 위탁자에게 발생한 직접 손해 배상
- 한도 = 본 DPA 직전 12개월 위탁 수수료의 100% (간접·결과 손해 = 한도 외)
- PIPA 위반 과징금 = 별도 (위탁자·수탁자 각자 책임)
